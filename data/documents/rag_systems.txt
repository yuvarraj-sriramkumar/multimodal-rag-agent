Retrieval-augmented generation (RAG) is a technique that enhances large language model outputs by grounding them in external, up-to-date knowledge retrieved at inference time. The core idea is to decouple factual knowledge storage from model parameters: rather than requiring the model to memorize every fact during training, RAG systems maintain a separate document store and retrieve the most relevant passages for each query, which are then concatenated to the prompt before generation. This approach addresses two fundamental weaknesses of standalone LLMs — knowledge cutoffs and hallucination — by supplying the model with verifiable, current evidence at query time.

A standard RAG pipeline consists of two main stages: offline indexing and online retrieval-generation. During indexing, source documents are split into smaller chunks (typically 200–1000 tokens), encoded into dense vector embeddings using a bi-encoder model such as sentence-transformers or OpenAI's text-embedding-ada-002, and stored in a vector database such as ChromaDB, Pinecone, Weaviate, or FAISS. Each chunk is stored alongside its original text and metadata (source file, page number, section heading). At query time, the user's question is encoded with the same embedding model, and an approximate nearest-neighbor search retrieves the top-k most semantically similar chunks. These chunks are formatted into a context block and prepended to the LLM's prompt, allowing the model to synthesize an answer grounded in the retrieved evidence.

Advanced RAG techniques go beyond naive single-stage retrieval. Hypothetical Document Embeddings (HyDE) generate a synthetic answer first and use its embedding for retrieval, improving recall for complex queries. Multi-hop RAG performs iterative retrieval, using intermediate answers to formulate follow-up queries. Re-ranking with a cross-encoder model re-scores the initial retrieval candidates for higher precision before passing them to the generator. Agentic RAG integrates retrieval as a tool within a reasoning loop, allowing the model to decide when to retrieve and what to query. Evaluation metrics for RAG systems include faithfulness (whether answers are grounded in retrieved context), answer relevance, and context precision/recall, often measured using frameworks like RAGAS or TruLens.
